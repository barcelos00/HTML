Estutura de um documento html

A estrutura de um documento html e definida pela declaração !DOCTYPE html e pleas tags estruturantes html, head e body.

A parte dos exercicios 1 a 4 foram a base para tudo porem feitos no bloco de notas, por isso foi preferido não adicionar eles aqui para ficar com um formatação visualmente bonita

Tag head

A tag head é utilizada para confighurações do navegador e consequentemente para configurações do documento html, as informações copntidas nestas tags não são visiveis, com exceção do titulo que ficara visivel no topo da pagina.

Tag body

A tag body é utilizada para exibir infromações para os usuarios, tudo que estiver dentro desta tag ficara visivel.

Nesta tag podemos ter outras tag que vâo ser renderizadas pelo navegador.

Tag HTML.

A tag HTML é a tag principal do documento HTML, ela é a tag raiz que contém as outras tags que possivbilitam a intepretação do documento HTML.

\instrução DOCTYPE.

A instrução DOCTYPE indica ao navegador qual a versão do HTML que iremos utilizar, ela não é uma tag, como dito ela é uma instrução que indica a versão do html 5.

Exemplo 8 Uso das Tags de Imagem e Vídeo

Este exemplo mostra diferentes formas de usar as tags <img> e <video>, incluindo o uso de <figure> e <figcaption> para agrupar mídias com legendas.

Descrição

As tags abordadas são:

  • <img>: Usada para inserir imagens na página.
  • <video>: Usada para inserir vídeos na página.
  • <figure>: Usada para agrupar mídias (imagens, vídeos, gráficos) com uma legenda.
  • <figcaption>: Usada para adicionar uma legenda ao conteúdo dentro de <figure>.

Este elemento agrua recursos de navegação do site, representando um grupo de links que consduzem as outras paginas da aplicação, ou seções da pagina corrente, correspondendo a area de "menus" em um aplicativo web

  • Portal do aluno CEUB
  • Angular
      Angular é um framework front-end criado pelo Google. Ele serve para desenvolver aplicações web de página única (SPA – Single Page Applications).
    • Diferente de usar apenas HTML, CSS e JavaScript puro, o Angular oferece:
      • Componentes: divide a aplicação em partes reaproveitáveis.
      • Data Binding: liga os dados do código à interface de forma automática.
      • Diretivas: adicionam comportamento extra aos elementos do HTML.
      • Roteamento: permite trocar de “páginas” sem recarregar o site todo.
      • TypeScript: o Angular é escrito em TypeScript (um superconjunto do JavaScript que adiciona tipagem).
  • Vue.js
      Framework progressivo para interfaces web, criado por Evan You (ex-Google). Focado em ser simples de aprender e leve
    • Caracteristicas:
      • Muito fácil de integrar em projetos já existentes.
      • Usa componentes também, mas de forma mais flexível que Angular.
      • Tem reatividade forte: quando os dados mudam, a interface muda automaticamente.
      • Tem reatividade forte: quando os dados mudam, a interface muda automaticamente.
  • Blazor C#
      Framework da Microsoft para criar aplicações web modernas. Com sua linguame principal sendo C#. Possui o foco de permitir que desenvolvedores .NET façam aplicações web sem precisar usar só JavaScript.
    • Diferente de usar apenas HTML, CSS e JavaScript puro, o Angular oferece:
      • Você escreve C# no lugar de JavaScript.
      • Funciona de dois modos:
        1. Blazor Server → o código roda no servidor e envia atualizações para o navegador.
        2. Blazor WebAssembly → o código C# roda direto no navegador usando WebAssembly.
      • Integra-se muito bem com ASP.NET e todo o ecossistema Microsoft.
